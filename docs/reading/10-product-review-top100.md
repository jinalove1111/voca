### 1-1. 9세 아동 (저학년, 읽기 속도 느림, 매일 사용)

**잘 되고 있는 것**: 히어로 CTA 1개가 화면을 지배하고("▶ 오늘의 학습 시작",
`Dashboard.jsx:209-223`) 보조 정보는 `<details>` 2개로 접혀 있어(:500, :573)
"어디를 누를지"는 3초 안에 성립한다. 실패 언어가 안전하고(오답에 벌점·질책
0), 잠긴 요소는 체크리스트가 아니라 "안개 속 한 줄"로 표현되며(`PaulTown.jsx:187`),
없는 기록을 지어내지 않는 정직성(폴의 기억/타임머신)이 일관된다. 세션 종료
카드의 "여기서 멈춰도 오늘 공부한 건 전부 저장돼 있어요"(`GuidedSession.jsx:219`)는
아동 정서 설계의 모범.

**마찰/혼란 (심각순)**:

1. **"← 나가기"가 사실은 로그아웃** — 다른 모든 화면의 "←"는 화면 이동인데
   홈의 "←"만 세션 삭제라(`Dashboard.jsx:373` → `App.jsx:783`), 뒤로가기로
   착각한 아이가 이름+PIN 재입력 벽에 부딪힌다. 로그인 세션 영속이라는 장점
   (`App.jsx:679, 778`)을 무효화하는 유일한 구멍.
2. **선물상자 직후 강제 시험, 출구 없음** — GiftReveal을 닫는 순간 오답
   스펠링 복습으로 자동 전환되고(`App.jsx:186-189`) SpellingReview에는
   뒤로가기 자체가 없어 큐를 다 비워야 탈출한다(`SpellingReview.jsx:50-77`).
   보상의 정점에서 정서가 급반전하는 구조.
3. **"복습"이 4종류로 병존** — 레벨업 미션(퀴즈 오답 3회 정답), 오늘
   오답노트+영구 큐(SpellingReview), 범위 필터 '복습 단어만'(합집합), 가이드
   세션 내 1회 재시도(`App.jsx:294-301, 660-673`, `LevelUpMission.jsx:14`,
   `GuidedSession.jsx:135-151`, `WordBrowser.jsx:17-22`). 이름·규칙·진입로가
   전부 달라 아이가 "복습이 뭔지" 하나의 그림을 만들 수 없다.
4. **거짓/기술적 텍스트** — "단어 공부" 버튼 부제가 하드코딩 "100개 단어"
   (`Dashboard.jsx:579`), 마이크 오류에 `err.name: err.message` 영어 원문 노출
   (`Dashboard.jsx:100-106`), 퀴즈 칭찬 풀에 "개꿀!" 슬랭(`QuizGame.jsx:9-13`).
5. **오답 화면이 너무 빨리 지나감** — 오답 시 1.8~2초 후 자동 전환
   (`WordDetail.jsx:450-454`, `QuizGame.jsx:314-319`) — 읽기 느린 아이는
   "정답: X"를 확인하기 전에 화면이 넘어간다.
6. **같은 개념의 다른 얼굴** — 4지선다 보기 번호가 화면마다 원문자/A~D로
   다르고(`WordDetail.jsx:476` vs `QuizGame.jsx:409`), 3x3 정원이 3곳에 존재

   `PaulTown.jsx:79-96`, `Dashboard.jsx:611-613`), "타임머신"이라는 이름이 두
   화면에 쓰인다(`GrowthAlbum.jsx:35-38` vs `TimeMachine.jsx:63-79`).

### 1-5. 횡단 진단 — 빠진 리텐션 장치

Duolingo 렌즈로 보면 이 앱은 **세션 설계(마이크로 세션, 완결감)는 이미 상위권**
이나, 리텐션의 나머지 두 축이 비어 있다: ① **돌아올 이유의 예고**(내일 무엇이
기다리는지 — 다음 모자/새싹/복습 예고가 세션 종료 카드에 없음,
`GuidedSession.jsx:188-223`), ② **부재 시 되부르는 채널**(알림/학부모 공유 0 —
웹앱이라 푸시가 어렵다면 PWA 설치+학부모 카톡 공유 텍스트가 무료 대안).
Khan Kids 렌즈로는 **텍스트 의존이 높다**(폴의 기억/안내문 전부 읽기 —
저학년에게 음성이 없음). Nintendo 렌즈로는 **"매일 하나의 눈에 보이는 변화"**
(별→씨앗→새싹)가 이미 있는데, 그 변화를 보러 가는 동선(홈 밴드 → 구경가기)이
학습 완료 순간과 연결돼 있지 않다 — 가장 감정이 고조된 순간(세션 완료)에 정원
변화를 보여주지 않고 홈으로만 보낸다.

또 하나의 구조적 도전: **"접기(details)로 단순화했다"는 가정**. 접기는 첫
화면 밀도를 낮췄지만(`Dashboard.jsx:495-606`), 모자/박물관/앨범/정원 같은 애착
장치 전부가 "더 많은 메뉴" 안에 숨어 발견성이 급감했다 — 관찰 레이어
(product_events)가 이 가설을 실측할 수 있는 첫 도구다.

### S티어 — 즉시 (마찰 제거 + 신뢰 회복, 대부분 문구·소규모 로직)

1. **[상/저/중] 홈 "← 나가기" 오터치 방지** — 라벨을 "로그아웃"으로 바꾸고 확인 1탭 추가. 매일 쓰는 화면의 유일한 파괴적 버튼이 가장 습관적인 위치에 있다(1-1-1).
2. **[상/저/중] SpellingReview에 "오늘은 여기까지" 탈출 버튼** — 큐는 유지되므로 학습 손실 0. 보상 직후 강제 시험+출구 없음은 내일 다시 열고 싶은 마음을 직접 깎는다(1-1-2).
3. **[상/저/상] 세션 완료 카드에 "내일 예고" 한 줄** — 어제 심은 별의 새싹/다음 모자까지 남은 진행/내일 복습 단어 수 중 하나를 파생 표시. 가장 감정이 높은 순간을 "내일 올 이유"로 연결(1-5). 전부 기존 파생 함수 재사용이라 저장 0.
4. **[상/저/하] "100개 단어" 하드코딩 → 실측 단어 수** — 거짓 정보는 아이보다 학부모·원장의 신뢰를 먼저 깎는다(`Dashboard.jsx:579`).
5. **[중/저/하] "개꿀" 등 슬랭 칭찬 문구 교체** — 학원 앱은 학부모 어깨너머 노출이 상수다(`QuizGame.jsx:9-13`).
6. **[중/저/하] 오답 화면 자동 전환을 "탭해서 계속"으로** — 정답 확인은 학습의 핵심 순간인데 1.8~2초는 읽기 느린 아이에게 짧다(1-1-5). 정답 시엔 현행 자동 전환 유지.
7. **[중/저/하] 가이드 세션 뒤로가기 라벨 수정** — "← 단어 목록"이 실제론 홈 이동(`WordDetail.jsx:584`). handoff에 이미 기록된 백로그.
8. **[중/저/하] 마이크 오류 문구 아동어 순화** — 영어 에러 원문은 콘솔로, 화면엔 "선생님께 이 화면을 보여주세요" 수준으로(`Dashboard.jsx:100-106`).
9. **[중/저/하] 4지선다 보기 번호 표기 통일** — 원문자/A~D 혼재는 같은 활동을 다른 활동처럼 보이게 한다(1-1-6).
10. **[중/저/하] 구버전 세션 안내 문구 분리** — 계정이 멀쩡한데 "계정을 찾을 수 없어요"로 불안 유발(`App.jsx:842`).
11. **[중/저/하] 칭찬 음성 재생 중 버튼 잠금 해제(스킵 허용)** — 빠르게 도는 학생의 누적 마찰 제거(`QuizGame.jsx:276-280`).
12. **[중/저/중] 유닛 셀렉터 오터치 확인** — 프로필 카드 안 select가 학습 목록을 통째로 바꾸는데 확인/되돌리기 안내가 없다(`Dashboard.jsx:407-423`).
13. **[중/저/하] SpellingQuestion 정답 공개 시 sad 이미지 → thinking 계열** — 유일하게 죄책감 신호가 남은 지점(`SpellingQuestion.jsx:331`).
