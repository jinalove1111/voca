# 08 — 문장 학습 데일리 미션 UX 설계 (Daily Mission UX for Sentence Learning)

작성일: 2026-07-23 · 작성: docs-maintainer · 상태: **설계 문서 — 코드 0줄, 스키마 변경 0**

이 문서는 문장 학습(Sentence Learning, v3.4 Phase A/B 구현 완료 상태)의
"데일리 미션" UX를 설계만 합니다. 구현/DDL/플래그 전환은 전부 비범위이며,
학생에게 보이는 모든 것은 기존 `readingStudentUI` 플래그(기본 false —
`SentencesTab` 게이팅과 동일) 뒤에 있어야 한다는 전제를 그대로 따릅니다.
구현 착수 전 운영자 승인 필요(저장소 헌법 규칙 12 — 학생 대상 기능은
승인된 트랙에서만).

## 1. 설계 원칙

1. **중복 금지.** 오늘의 미션 4칸에 5번째 바를 추가하지 않는다(§6.1).
   GuidedSession(단어 리추얼)을 대체·변형하지 않는다 — 문장 미션은
   **문장 도메인의 병렬 리추얼**이다.
2. **보상 없음.** SentenceLearningFlow의 승인 스펙("보상/재화/게임화
   없음")을 미션 레벨에도 그대로 적용 — 별/XP/티켓/스티커 지급 0.
   완료의 보상은 완료 화면의 축하와 진행 표시뿐.
3. **스트릭 징벌 금지.** 스트릭은 이어질 때만 따뜻하게 표시하고, 끊김을
   경고·암시·소급 표시하지 않는다(§5).
4. **홈 화면 불변.** Dashboard에 새 카드를 추가하지 않는다(홈 ≥80% 불변
   원칙, Dashboard.jsx 주석 실측). 미션 UI는 전부 문장 탭 내부.
5. **새 저장 최소화.** 오늘/주간 진행은 `sentence_progress`의
   `mastered_at` 날짜에서 파생 계산한다(새 테이블 없이 가능). 파생
   불가능한 값은 정직하게 표시하지 않는다(§5.2).
6. **한 화면 = 한 활동**(SentenceLearningFlow 아동 UX 원칙 유지).

## 2. 미션 정의

- **오늘의 문장 미션 = 오늘 핵심 문장 N개 마스터** (N 기본 1, 최대 3 —
  핵심 문장이 N개 미만 남았으면 남은 수가 곧 목표. 남은 핵심 문장이
  0이면 미션 카드 대신 "전부 완료" 상태 §3.4-d).
- 대상 문장 선택: 현재 유닛의 핵심 문장 중 미마스터 문장을 중요도
  (`importance_level` 내림차순) → 지문 내 순서로 결정론 선택. 무작위
  없음(엔진 관례).
- 오늘 판정: `mastered_at`의 로컬 날짜 == 오늘. (이미 마스터한 문장을
  복습해도 `mastered_at`은 최초 1회만 기록되므로 미션에 이중 계산되지
  않음 — `applyStageResult` 실측.)

## 3. 화면 설계 + 와이어프레임

### 3.1 미션 시작 화면 (문장 탭 상단 카드)

`SentencesTab` 지문 아코디언 **위**에 카드 1장 추가. 흰 `rounded-3xl`
카드, CTA만 보라/핑크 그라데이션.

```
┌─────────────────────────────────────────┐
│  📜 오늘의 문장 미션          ● ○ ○     │  ← 진행 점 = 오늘 목표 N개
│                                         │
│  오늘은 이 문장을 마스터해봐요:          │
│  "I like to read books."               │
│  ★★★★ 자주 출제 · 핵심 문장           │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │      ▶ 오늘의 문장 시작하기        │  │  ← from-purple-500 to-pink-500
│  └───────────────────────────────────┘  │     min-h 44px, font-black
│  이번 주  월 화 수 목 금 토 일           │
│           ●  ●  ○  ◎  ·  ·  ·          │  ← §4 주간 진행(◎=오늘)
└─────────────────────────────────────────┘
```

- 문구: 시작 전 부제는 폴 톤 한 줄 — "지금 문장 하나만 마스터하면
  돼요!"(SentenceLearningFlow 헤더 문구 재사용, 새 문구 불필요).
- 진행 점: 오늘 목표 N개 = 점 N개. 채움 ●(오늘 마스터), 빈 ○(남음).
  SentenceLearningFlow의 6단계 점과 혼동되지 않게 이 카드에서는 "문장
  단위" 점임을 라벨 없이도 위치(카드 헤더 우측)로 구분.

### 3.2 미션 진행 중 (기존 풀스크린 플로우 + 얇은 미션 컨텍스트)

학습 자체는 기존 `SentenceLearningFlow` 그대로(6단계 점, STAGE_PRAISE,
ENCOURAGE, 적응 지원). 추가는 상단 컨텍스트 한 줄뿐 — GuidedSession의
sticky 헤더 관례(`bg-white/95 backdrop-blur border-b-2 border-purple-100`).

```
┌─────────────────────────────────────────┐
│ 📜 오늘의 문장 1 / 3        ● ○ ○      │  ← sticky, 문장 단위 진행
├─────────────────────────────────────────┤
│        (기존 SentenceLearningFlow       │
│         화면 그대로 — 단계 점 6개,       │
│         문장 본문 text-2xl, 44px 버튼)  │
└─────────────────────────────────────────┘
```

- 중도 이탈: 언제든 닫기 가능. 진행은 기존 fire-safe upsert가 이미
  저장하므로 "이어하기"는 `current_stage`에서 자동 재개(기존 동작) —
  미션 상태도 파생값이라 잃는 것이 없다.

### 3.3 미션 완료 화면

N개째 문장이 mastered 되는 순간, 기존 문장별 축하 카드 대신 미션 완료
카드를 1회 표시. GuidedSession 완료 화면 관례 그대로: 배경
`from-indigo-500 to-purple-600` 풀스크린, 흰 `rounded-3xl` 카드,
HeroReaction(성공 이미지 재사용 — 새 리소스 없음).

```
╔═════════════════════════════════════════╗
║   (indigo→purple 그라데이션 배경)        ║
║  ┌───────────────────────────────────┐  ║
║  │        [폴 성공 이미지]            │  ║
║  │   오늘의 문장 미션 완료! 🎉        │  ║
║  │  "미션 완료! 오늘도 약속을         │  ║
║  │   지켰네요." (07 문서 E 카테고리)  │  ║
║  │                                   │  ║
║  │        ● ● ●   오늘 3 / 3         │  ║
║  │                                   │  ║
║  │  ┌─────────────────────────────┐  │  ║
║  │  │   🏠 오늘은 여기까지         │  │  ║  ← 주 CTA(그라데이션)
║  │  └─────────────────────────────┘  │  ║
║  │  ┌─────────────────────────────┐  │  ║
║  │  │   ▶ 한 문장 더 해보기        │  │  ║  ← 남은 핵심 문장 있을 때만
║  │  └─────────────────────────────┘  │  ║     (테두리형 보조 버튼)
║  │  여기서 멈춰도 오늘 공부한 건      │  ║
║  │  전부 저장돼 있어요!              │  ║  ← GuidedSession 안심 문구 관례
║  └───────────────────────────────────┘  ║
╚═════════════════════════════════════════╝
```

- "한 문장 더"는 미션 목표를 늘리지 않는다 — 추가 학습은 자유 학습이며
  별도 표기/보상 없음(완료 상태 유지). "더 해야 한다"는 압박 프레임을
  만들지 않기 위해 보조 버튼 톤(테두리형)으로만 둔다.
- 완료 문구는 `docs/reading/07-encouragement-messages.md` E 카테고리에서
  결정론 선택(엔진 무작위 금지 관례 — 날짜 시드 등).

### 3.4 오늘의 진행 (상태별 카드 변형)

시작 카드(§3.1)가 상태에 따라 내용만 바뀐다 — 카드 수는 항상 1장.

- (a) **미시작**: §3.1 그대로. `● ○ ○` 전부 빈 점 + 시작 CTA.
- (b) **진행 중**(오늘 1개 이상 마스터, 목표 미달): 점 일부 채움, CTA
  라벨 "▶ 이어서 하기". 부제: "거의 다 왔어요!" 계열(07 문서 C).
- (c) **완료**: 점 전부 채움 + "오늘의 문장 미션 완료! ✓"(초록
  `bg-green-100 text-green-600` 칩 — 오늘의 미션 4/4 배지 관례). CTA는
  "📖 자유롭게 더 읽기"(문장 목록 스크롤 유도, 압박 없음).
- (d) **유닛 핵심 문장 전부 마스터**: 미션 카드 대신 정직한 완료 상태 —
  "이 유닛의 핵심 문장을 전부 마스터했어요! 새 지문이 오면 미션이 다시
  시작돼요." (없는 목표를 만들어내지 않음 — paulMemory 진실성 원칙.)

### 3.5 상태 전이

```
[미시작 idle]
   │ 오늘 첫 마스터(sentence_progress.mastered_at == 오늘)
   ▼
[진행 중 in_progress]  (오늘 마스터 수 1..N-1)
   │ 오늘 마스터 수 == N
   ▼
[완료 completed] ── 추가 학습해도 상태 불변(자유 학습)
   │ 자정(로컬 날짜 변경) → 파생값이 자연히 리셋
   ▼
[미시작 idle] (다음 날) 또는 [전부 완료 all_mastered] (남은 핵심 문장 0)
```

전부 파생 상태(저장 없음)라 "미션 상태 꼬임" 버그가 구조적으로 불가능
— 자정 전이도 별도 코드 없이 날짜 계산 결과가 바뀔 뿐이다.

## 4. 주간 진행

- 표시: 이번 주(월~일) 7칸 점. `●` = 그날 마스터한 핵심 문장 ≥ 1
  (`mastered_at` 날짜 파생), `◎` = 오늘(진행 중), `○` = 지나갔지만 기록
  없는 날, `·` = 아직 오지 않은 날.
- **징벌 금지 적용**: `○`는 중립 회색(`text-gray-300`)만 — 빨간색/❌/
  "놓쳤어요" 라벨 절대 금지. 지나간 빈 날에 대한 어떤 문구도 붙이지
  않는다. 주간 요약 문구는 채워진 날에 대해서만 말한다("이번 주 3일
  문장을 마스터했어요!") — 빈 날 개수는 절대 언급하지 않는다.
- 한계(정직): `mastered_at`은 문장당 최초 1회만 기록되므로, "그날 이미
  마스터한 문장을 복습만 한 날"은 점이 채워지지 않는다. v1은 이 한계를
  받아들이고(마스터 기준 주간 표시라고 카드에 명시하지 않되 과장도
  하지 않음), 복습까지 반영하려면 일자별 기록 저장이 필요하다(§6.4).

## 5. 스트릭 표시

### 5.1 표시 규정 (징벌 금지 — paulMemory 원칙 상속)

- 이어지고 있을 때만 표시: "🔥 3일째 문장과 만나는 중이에요" — 3일
  이상부터만 노출(paulMemory `streak-going` 임계 관례). 1~2일은 표시
  자체를 생략(숫자가 작다고 초라해 보이게 하지 않음).
- **금지 목록**: "스트릭이 끊겨요!", "오늘 안 하면 사라져요", 끊긴 후
  "0일" 표시, 끊김 직후 회색 불꽃 등 상실 시각화, 푸시/배지로 스트릭
  유지 독촉. 전부 금지.
- 끊긴 후 복귀: 스트릭 언급 대신 환영만 — paulMemory `comeback` 템플릿
  ("N일 만이네요! 다시 만나서 정말 반가워요") 프레임 재사용. 새 스트릭
  시작은 조용히 다시 세기 시작할 뿐 "다시 시작이네요" 같은 소급 언급도
  하지 않는다.

### 5.2 데이터 정직성 (스키마 실측에 따른 v1 결정)

`sentence_progress.last_practiced_at`은 문장당 **최신 시각 하나**만
유지하므로, 문장 전용 연속 학습일을 현행 스키마로 정확히 파생할 수
없다(어제의 연습 시각이 오늘 연습으로 덮여 사라짐). 없는 데이터를
계산하는 척하지 않는다(paulMemory 진실성 원칙). 따라서:

- **v1(권장)**: 문장 전용 스트릭을 만들지 않고, 문장 미션 카드에는
  스트릭을 표시하지 않는다. 스트릭은 기존 단어 학습 스트릭(Dashboard
  🔥 배지)이 유일한 스트릭으로 남는다 — 학생에게 스트릭이 2종류가 되는
  혼란도 없다.
- **v2(선택, 운영자 결정 필요)**: `mastered_at` 날짜 기반의 "마스터
  스트릭"(연속으로 마스터가 있었던 날)은 현행 스키마로 파생 가능 —
  단, 복습만 한 날 끊긴 것처럼 보이는 한계가 §4와 동일하게 존재.
  일자별 기록 테이블 신설은 §6.4의 DDL 절차를 따른다.

## 6. 기존 시스템과의 관계 및 확장 지점

### 6.1 오늘의 미션 4칸 — 편입하지 않는 이유(v1)

- 4칸은 단어 도메인 지표(`dailyProgress.words/examples/quizzes/
  pronunciations`)에 결합돼 있고, 4/4 완료가 티켓
  (`ticketEconomy.js` `daily-mission-complete`)·XP(`paulRankShared.js`
  일별 기간키)·선물상자 경제와 직접 연결된다. 문장 지표를 5번째 칸으로
  넣으면 이 경제 전체의 재검토가 필요하다(XP 파밍 사고 이력 —
  `paulRankShared.js` 주석 실측). 문장 미션은 보상 0 원칙이므로 경제와
  분리된 별도 카드가 안전하다.
- **확장 지점**: 이후 편입을 원하면 ① `XP_EVENT_TABLE`의
  `status: 'planned'` 예약 슬롯 관례(서버가 무조건 거부)로 이벤트를
  먼저 예약하고 ② 운영자 승인 후 별도 세션에서 경제 검토와 함께 전환.

### 6.2 3분 데일리 리추얼(GuidedSession)

- 역할 분담: GuidedSession = 오늘의 **단어** 전체를 마이크로 세션으로
  완주. 문장 미션 = 오늘의 **핵심 문장 N개** 마스터. 서로 호출하지도
  대체하지도 않는다.
- 재사용하는 것은 코드가 아니라 **UX 관례**: 명확한 완료 순간, 완료
  카드의 "여기까지/계속" 이중 선택, "저장돼 있어요" 안심 문구, sticky
  진행 헤더. `dailyRitual.js`의 `planSessionSize`는 단어량 계획용이라
  목표 1~3개인 문장 미션에는 쓰지 않는다.
- **확장 지점**: 추후 "오늘의 단어를 끝낸 학생에게 문장 미션을 이어서
  제안"(GuidedSession 완료 카드에 진입 버튼 1개)은 자연스러운 연결점
  — 단, 홈/완료 카드 문구가 압박 프레임("이것도 해야 끝나요")이 되지
  않도록 보조 버튼 톤으로만.

### 6.3 격려 문구 엔진

각 상태의 문구는 `07-encouragement-messages.md`의 카테고리와 1:1로
매핑한다: 정답 → A, 오답 재시도 → B(기존 ENCOURAGE 3종의 확장 풀),
목표 근접(N-1개째 완료 등) → C, 문장 mastered → D, 미션 완료 화면 → E.
선택은 결정론(시드: 날짜+문장 id 등 — 엔진 무작위 금지 관례).

### 6.4 데이터/DDL 경계

- v1은 **새 테이블/컬럼 0** — 전부 `sentence_progress` 파생.
- 일자별 학습 기록(§4 한계, §5.2 v2)이 필요해지는 시점에는 저장소 헌법
  규칙 8/9대로 `supabase_v{n}_*.sql` 멱등 마이그레이션 파일만 준비하고
  운영자가 수동 실행, 클라이언트는 테이블 부재 시 폴백(현행
  `sentenceProgressApi` 관례).

## 7. 비범위 (Non-goals)

- 코드/플래그/스키마 변경 일절 없음(이 문서는 설계만).
- 보상·재화·랭킹·경쟁 요소 없음(운영자 승인 스펙 유지).
- 홈(Dashboard) 화면 변경 없음.
- 푸시 알림/리마인더 없음 — 특히 스트릭 유지 독촉 알림은 설계상 금지.
- 학부모/관리자 화면 변경 없음(문장 미션 통계 노출은 별도 설계 필요).
